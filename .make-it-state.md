## Phase F2 + G -- What Was Done (2026-03-19)

### New Files Created

| File | Purpose |
|------|---------|
| `src/lib/ai/sanitize.ts` | Input sanitization: strips injection patterns, wraps user input in `<user_input>` delimiter tags, validates prompt/document size |
| `src/lib/ai/validate.ts` | Output validation: schema-based checks for AI responses, safe JSON parsing (replaces raw JSON.parse) |
| `src/lib/ai/rate-limit.ts` | Per-user rate limiting: sliding window on requests/minute and tokens/minute for AI endpoints |
| `src/lib/ai/pii-masker.ts` | PII pseudonymization: masks emails, phones, SSNs before AI submission, restores in response |
| `src/lib/ai/errors.ts` | Error sanitization: maps provider errors to safe generic messages (no endpoint/model/key leakage) |
| `src/lib/ai/safety-preamble.ts` | Immutable safety preamble: anti-injection/anti-jailbreak instructions prepended to every system prompt |
| `src/lib/ai/validate-template.ts` | Prompt template validation: blocklist, renderPromptSafe, testPromptDraft, risk flag detection |

## Tech Stack

- Next.js 16.1.6 (monolith with API routes)
- Prisma ORM with PostgreSQL
- OIDC authentication with stateless JWT sessions
- Database-driven RBAC (4 system roles, 44 ADMIN / 32 ANALYST / 8 VIEWER / 11 VENDOR permissions)
- 6 AI Agents: VERA, CARA, DORA, SARA, RITA, MARS
- Multi-provider AI (Azure AI Foundry primary, Claude/OpenAI/Ollama alternatives)
- AI Operational Safety: input sanitization, output validation, rate limiting, PII masking, error sanitization, system prompt hardening
- Prompt Template Validation: content blocklist, immutable safety preamble, draft/test/publish workflow, risk_flag audit logging
- Docker Compose with mock-oidc
- Radix UI + Tailwind CSS
- Security headers (CSP, HSTS, X-Frame-Options)
- Terraform (Azure: PostgreSQL, Key Vault, ACR, App Service)

## AI Safety Controls Summary

| Control | Module | What It Does |
|---------|--------|-------------|
| Input Sanitization | `lib/ai/sanitize.ts` | Strips injection patterns, wraps user input in `<user_input>` tags |
| Output Validation | `lib/ai/validate.ts` | Schema-based checks on AI responses, safe JSON parsing |
| Rate Limiting | `lib/ai/rate-limit.ts` | Per-user request and token budgets on all AI endpoints |
| PII Masking | `lib/ai/pii-masker.ts` | Pseudonymizes emails, phones, SSNs before AI; restores after |
| Error Sanitization | `lib/ai/errors.ts` | Maps provider errors to generic safe messages |
| Safety Preamble | `lib/ai/safety-preamble.ts` | Anti-injection/jailbreak instructions on every system prompt |
| Prompt Size | `lib/ai/sanitize.ts` | Rejects prompts/documents exceeding configurable limits |
| Template Validation | `lib/ai/validate-template.ts` | Blocklist on admin prompt saves, renderPromptSafe, testPromptDraft |
| Draft/Publish | API + Schema | Prompts saved as draft, must pass tests before publishing |
| Audit Logging | PromptAuditLog | risk_flag on suspicious edits for security review |
